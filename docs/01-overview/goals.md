# 目的・解決する課題

> Status: Draft
> 最終更新: 2026-01-28

本ドキュメントは、Hotateプロジェクトの目的と解決する課題を明確にする。

---

## プロジェクトの目的

Hotateの目的は、モバイル端末からVPS・ホームサーバーへ日本語IME対応のSSH接続を提供し、外出先での緊急サーバー操作を可能にすることである。

---

## 解決する課題

### 課題一覧

| # | 課題 | 影響 | 優先度 |
|---|------|------|--------|
| 1 | モバイルSSHアプリのIME非対応 | 日本語入力が必要なコマンドやファイル編集ができない | High |
| 2 | 複数サーバーの接続情報管理 | 接続先が増えるたびに設定の手間が発生する | Medium |
| 3 | 特殊キー入力の困難さ | モバイルキーボードではCtrl+C等の送信が困難 | High |

### 課題の詳細

#### 課題1: モバイルSSHアプリのIME非対応

| 項目 | 内容 |
|------|------|
| 現状 | モバイルSSHクライアント（Termius等）はIMEとの連携が不安定で、日本語入力が表示されない |
| 問題点 | 日本語ファイル名の操作やgitコミットメッセージ入力が実質不可能 |
| 解決策 | HTML `<input>` のcompositionイベントで変換確定を検知し、確定後のみSSHストリームへ送信する |

#### 課題2: 複数サーバーの接続情報管理

| 項目 | 内容 |
|------|------|
| 現状 | 接続先ごとにホスト名・ポート・認証方法を毎回入力する必要がある |
| 問題点 | 緊急時に素早く接続できない |
| 解決策 | ホスト情報をサーバー側JSONファイルに保存し、ワンタップで接続できるようにする |

#### 課題3: 特殊キー入力の困難さ

| 項目 | 内容 |
|------|------|
| 現状 | モバイルのソフトウェアキーボードにはCtrl, Tab, Escキーが存在しない |
| 問題点 | プロセスの中断（Ctrl+C）やタブ補完が使えず、サーバー操作効率が著しく低下する |
| 解決策 | 画面上に特殊キーツールバーを常設し、タップで対応するエスケープシーケンスを送信する |

---

## 成功基準

| 基準 | 指標 | 目標値 |
|------|------|--------|
| SSH接続の確立 | パスワード/Key認証で接続成功 | 両方式で動作 |
| 日本語入力 | IME変換→確定→送信の一連の動作 | 変換途中の文字が送信されない |
| モバイル操作性 | 特殊キーのタップ送信 | Tab, Ctrl+C, Ctrl+D, Esc等が動作 |
| PWA | ホーム画面から起動 | 静的リソースがキャッシュされ即座に表示 |

---

## 非目標

以下は本プロジェクトの目標ではない：

| 非目標 | 理由 |
|--------|------|
| マルチユーザー対応 | 個人利用のため。Basic認証で十分 |
| ファイル転送（SCP/SFTP） | ターミナル操作のみに集中する |
| SSH鍵の生成・管理UI | サーバー側に既存の鍵をマウントする方式で対応 |
| セッション永続化・再接続 | 初期スコープでは切断時に新規接続とする |
